# Using Brightspace Pulse

Brightspace Pulse is a meaningful part of the public learner-help surface. Including it in the demo makes the help center feel much closer to the real D2L ecosystem instead of stopping at browser login and release updates.

## What learners use Pulse for

- viewing upcoming work and deadlines
- checking courses and course content
- reviewing grades
- managing push notifications
- switching between multiple accounts when needed

## When to route learners here

Use a Pulse-focused page when the question is about:

- reminders and mobile notifications
- seeing work across courses in one mobile view
- viewing grades or content on a phone
- troubleshooting app behavior or account switching

## Core support framing

Pulse should feel like an extension of the learner experience, not a separate product silo. The help pattern is:

1. confirm the learner can access the right Brightspace account
2. confirm they are using the expected mobile app flow
3. route app-specific issues to notification, account, or troubleshooting guidance
